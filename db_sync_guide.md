# คู่มือการซิงค์ฐานข้อมูลสำหรับการทำงานข้ามเครื่อง 🔄

เอกสารนี้แนะนำขั้นตอนการซิงค์ฐานข้อมูล MySQL ระหว่างเครื่องคอมพิวเตอร์ของคุณผ่าน GitHub โดยใช้สคริปต์อัตโนมัติที่มีอยู่ในโปรเจกต์

---

## 📥 ขั้นตอนที่ 1: ก่อนปิดงาน (เครื่องต้นทางที่มีข้อมูลล่าสุด)

เมื่อต้องการหยุดทำงานจากเครื่องปัจจุบันและย้ายไปเครื่องอื่น ให้ทำตามนี้:

1. **ส่งข้อมูล MySQL ออกเป็นไฟล์ SQL:**
   * ดับเบิลคลิกไฟล์ [export_db.bat](file:///d:/cnpapp/export_db.bat) ในโฟลเดอร์โปรเจกต์
   * ระบบจะดึงข้อมูลใน MySQL ออกมาเขียนทับไฟล์ `admin_cnpapp (3).sql` โดยอัตโนมัติ
2. **ส่งไฟล์ขึ้น GitHub:**
   * เปิด VS Code ไปที่แถบ **Source Control**
   * ตรวจสอบว่ามีไฟล์ `admin_cnpapp (3).sql` เปลี่ยนแปลง
   * กด **Sync Changes** (หรือพิมพ์ `git commit` และ `git push`) เพื่อส่งไฟล์ฐานข้อมูลล่าสุดขึ้น GitHub

---

## 📤 ขั้นตอนที่ 2: เมื่อเริ่มงาน (เครื่องปลายทางที่ต้องการเขียนต่อ)

เมื่อเปิดคอมพิวเตอร์อีกเครื่องขึ้นมาทำงาน ให้ทำตามนี้:

1. **ดึงไฟล์ล่าสุดจาก GitHub:**
   * กด **Sync Changes** ใน VS Code (หรือรัน `git pull origin master`) เพื่อดึงโค้ดและไฟล์ SQL ล่าสุดลงมา
2. **นำข้อมูลเข้า MySQL ในเครื่อง:**
   * ดับเบิลคลิกไฟล์ [import_db.bat](file:///d:/cnpapp/import_db.bat) ในโฟลเดอร์โปรเจกต์
   * ระบบจะล้างฐานข้อมูลเก่าในเครื่องนั้น และนำเข้าข้อมูลล่าสุดจากไฟล์ `admin_cnpapp (3).sql` ให้ทันที
3. **เริ่มทำงาน:**
   * ดับเบิลคลิกไฟล์ [start_all.bat](file:///d:/cnpapp/start_all.bat) เพื่อรันระบบแล้วพัฒนาต่อได้เลย!

---

> [!WARNING]
> **ข้อควรระวัง:** 
> การรัน [import_db.bat](file:///d:/cnpapp/import_db.bat) จะทำการเขียนทับ (Overwrite) ฐานข้อมูลทั้งหมดในเครื่องปลายทาง ดังนั้นหากมีข้อมูลทดสอบใหม่ๆ ที่ยังไม่ได้กด Export จากเครื่องปลายทาง ข้อมูลนั้นจะสูญหาย โปรดตรวจสอบให้แน่ใจว่าทำตามลำดับขั้นตอนเสมอ
