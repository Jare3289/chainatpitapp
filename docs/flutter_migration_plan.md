# แผนเตรียมการพัฒนา CNP APP สู่ Flutter (Multi-platform App & Web)

เอกสารฉบับนี้เป็นแผนงานและข้อมูลอ้างอิงสำหรับการย้ายส่วนติดต่อผู้ใช้ (Frontend Migration) ของระบบ CNP APP ไปเป็น **Flutter** เพื่อรองรับการทำงานในลักษณะ Multi-platform (เว็บแอปพลิเคชัน, แอป Android บน Play Store, และแอป iOS บน App Store) โดยยังรักษาระบบหลังบ้าน (Backend) และฐานข้อมูลเดิมไว้

---

## 📌 สรุปหลักการของระบบใหม่ (Architecture Concept)

```
[ Frontend: เขียนด้วย Flutter (โค้ดชุดเดียว) ]
  ├── 🌐 Web App (เปิดผ่านคอมพิวเตอร์/เบราว์เซอร์)
  ├── 🤖 Android App (โหลดผ่าน Google Play Store)
  └── 🍏 iOS App (โหลดผ่าน Apple App Store)
          ↓ (ยิง HTTP Request / JSON)
[ Backend: PHP 8.0+ (โค้ดระบบเดิมบน Server) ]
  └── 🗄️ Database: MySQL / MariaDB (ก้อนเดิม)
```

---

## 🛠️ จุดที่ต้องเตรียมปรับปรุงระบบเดิม (Backend API Prep)

เพื่อให้ PHP Backend เดิมคุยกับ Flutter ได้อย่างราบรื่น ต้องทำการปรับปรุงดังนี้:

1. **การตั้งค่า CORS (Cross-Origin Resource Sharing):**
   * เพื่ออนุญาตให้แอปพลิเคชันจาก Domain อื่น หรือ Native App เข้าถึง API ได้
   * เพิ่ม Header ต่อไปนี้ในจุดเริ่มต้นของการเรียกใช้งาน API (เช่นในไฟล์กลุ่ม `api/`):
     ```php
     header("Access-Control-Allow-Origin: *");
     header("Access-Control-Allow-Headers: Content-Type, X-CSRF-Token, Authorization");
     header("Access-Control-Allow-Methods: GET, POST, DELETE, OPTIONS");
     ```

2. **การจัดการ Authentication:**
   * Flutter สามารถใช้ Package `dio_cookie_manager` เพื่อจัดเก็บ Session Cookie (`PHPSESSID` และ `cnp_auth`) ได้เหมือนเว็บบราวเซอร์ปกติ
   * หรือเตรียมแผนการอัปเกรดระบบ Auth เป็น Token-Based (เช่น JWT หรือ Bearer Token) ในอนาคตหากต้องการความปลอดภัยและมาตรฐานที่ดียิ่งขึ้น

3. **การจัดการ CSRF Token:**
   * ดึง token จาก `api/csrf.php` หรือ `api/me.php` มาเก็บไว้ฝั่ง Flutter
   * แนบ Header `X-CSRF-Token` ไปกับคำขอ POST/DELETE เสมอ

## 📸 แผนการพัฒนาระบบสแกนใบหน้า (Facial Recognition)

การพัฒนาฟังก์ชันสแกนใบหน้าเมื่อเดินผ่านกล้อง สามารถเลือกวิธีพัฒนาได้ 2 แนวทาง:

### แนวทางที่ 1: ใช้ Cloud AI (แนะนำเพื่อความแม่นยำและรวดเร็ว)
* **เครื่องมือ:** AWS Rekognition, Google Cloud Vision หรือ Azure Face API
* **หลักการทำงาน:**
  1. กล้องในแอป Flutter จับภาพใบหน้าเมื่อตรวจพบการเคลื่อนไหว
  2. แอปส่งรูปภาพใบหน้าผ่าน API ไปที่ Cloud AI
  3. Cloud AI ตรวจสอบเปรียบเทียบกับภาพต้นแบบในคลัง และคืนค่ารหัสผู้ใช้งาน (User ID) กลับมา
  4. ระบบทำการบันทึกเวลาเข้าใช้งานลงในฐานข้อมูล PHP/MySQL

### แนวทางที่ 2: ใช้สคริปต์ Python ใน Server (ฟรีค่าบริการคลาวด์)
* **เครื่องมือ:** OpenCV + `face_recognition` library ในฝั่งเซิร์ฟเวอร์
* **หลักการทำงาน:**
  1. เก็บข้อมูลใบหน้าพนักงาน/นักเรียนเป็นค่าตัวเลข (Vector/Embedding) ไว้ในฐานข้อมูล
  2. เมื่อแอป Flutter จับภาพได้ จะยิงรูปภาพมาที่ Python API ในเซิร์ฟเวอร์
  3. สคริปต์ Python คำนวณความคล้ายคลึงของใบหน้า (Face Distance)
  4. ส่งผลการระบุตัวตนกลับไปที่ฐานข้อมูลหลัก

## 🍏 การพัฒนาและขึ้นระบบสำหรับ iOS (โดยไม่ต้องมีเครื่อง Mac ส่วนตัว)

การสร้างและทดสอบแอป iOS มีข้อกำหนดพิเศษจาก Apple แต่สามารถบริหารจัดการได้ด้วยวิธีดังนี้:

1. **ใช้เครื่อง Mac ของเพื่อน:**
   * เขียนโค้ดระบบด้วย Windows ตามปกติ และนำโค้ดขึ้น Git (เช่น GitHub)
   * เมื่อต้องการ Build หรือนำขึ้นทดสอบ (TestFlight/App Store) ให้ไปดึงโค้ดลงมาที่เครื่อง Mac ของเพื่อนเพื่อทำเรื่องคอมไพล์แอปผ่าน Xcode
2. **ใช้บริการ Cloud Mac (CI/CD):**
   * ใช้บริการผู้ช่วยคอมไพล์โค้ดอย่าง **Codemagic** หรือ **GitHub Actions**
   * โควต้าฟรีของ Codemagic (500 นาที/เดือน) หรือ GitHub Actions สำหรับคลังข้อมูลส่วนตัว (200 นาที Mac/เดือน) เพียงพอต่อการ Build เพื่อส่งแอปเข้า Store โดยไม่จำเป็นต้องใช้เครื่อง Mac ตัวจริง
