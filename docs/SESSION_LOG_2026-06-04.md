# Session Log: 4 มิถุนายน 2026

เอกสารฉบับนี้เป็นการสรุปภาพรวมของการดำเนินการ, การแก้ไขปัญหา และการพัฒนาระบบ (CNP APP) ประจำวันที่ 4 มิถุนายน 2026

## 1. การปรับปรุงฐานข้อมูลและการทำให้ข้อมูลเป็นมาตรฐาน (Data Standardization)

พบปัญหาความไม่สม่ำเสมอของการบันทึกชื่อกลุ่มสาระการเรียนรู้ ซึ่งส่งผลต่อการดึงข้อมูล (Query) และการแสดงผลบนหน้าเว็บไซต์:
- **ปัญหา:** มีการบันทึกทั้งรูปแบบ `"สังคมศึกษาศาสนาและวัฒนธรรม"` (เขียนติดกัน) และ `"สังคมศึกษา ศาสนา และวัฒนธรรม"` (เว้นวรรค)
- **การแก้ไข:** 
  - สร้างสคริปต์อัตโนมัติ `api/update_dept_name.php` เพื่อทำการสแกนตารางทั้งหมดในฐานข้อมูล
  - ดำเนินการแทนที่คำว่า `"สังคมศึกษาศาสนาและวัฒนธรรม"` ให้เป็น `"สังคมศึกษา ศาสนา และวัฒนธรรม"` ทั้งหมด (อัปเดตข้อมูลสำเร็จ 67 รายการในตาราง `subjects` และ `teachers`) เพื่อให้ข้อมูลในระบบเป็นมาตรฐานเดียวกัน 100%

## 2. การพัฒนาระบบแสดงผลบุคลากร (Teacher Data Sorting)

ปรับปรุงระบบการจัดเรียง (Sorting Logic) ในหน้าจอแสดงผลข้อมูลครูและบุคลากร (`views/admin_teachers.html`) ให้สอดคล้องกับโครงสร้างและลำดับอาวุโสทางราชการอย่างถูกต้อง โดยจัดลำดับความสำคัญ (Priority) เรียงตามเงื่อนไขดังต่อไปนี้:

1. **ระดับผู้บริหารกลุ่มสาระ (Department Position):**
   - ผู้ที่ดำรงตำแหน่ง **"หัวหน้ากลุ่มสาระการเรียนรู้"** จะถูกจัดให้อยู่บนสุดเสมอ
   - ตามด้วย **"รองหัวหน้ากลุ่มสาระการเรียนรู้"**

2. **ระดับตำแหน่งงาน (Position):**
   - จัดเรียงตามสถานภาพการทำงาน: ผู้อำนวยการ -> รองผู้อำนวยการ -> **ครู** -> **ครูผู้ช่วย** -> พนักงานราชการ -> **อัตราจ้าง** -> **นักศึกษาฝึกสอน** 
   - *(หมายเหตุ: การเรียงลำดับนี้ทำให้ครูอัตราจ้าง และนักศึกษาฝึกสอน ถูกจัดลำดับต่อท้ายกลุ่มพนักงานประจำเสมอ)*

3. **ระดับวิทยฐานะ (Academic Standing):**
   - ในกรณีที่ตำแหน่งงาน (Position) อยู่ในระดับเดียวกัน (เช่น เป็น "ครู" เหมือนกัน) ระบบจะจัดเรียงตามระดับวิทยฐานะจากสูงไปต่ำ:
   - เชี่ยวชาญพิเศษ -> เชี่ยวชาญ -> ชำนาญการพิเศษ -> ชำนาญการ -> ไม่มีวิทยฐานะ

4. **รหัสประจำตัว (Teacher ID):**
   - หากความสำคัญในข้อ 1-3 เท่ากันทั้งหมด ระบบจะใช้ **รหัสประจำตัวครู (IDT)** ในการเรียงลำดับ (จากเดิมที่ใช้ชื่อภาษาไทย) เพื่อความเป็นระเบียบและค้นหาได้ง่ายขึ้น

## 3. การปรับปรุงระบบแจ้งเตือนให้ฉลาดขึ้น (Notification Optimization)

พบปัญหาว่าระบบแจ้งเตือน (Notifications) แสดงผลซ้ำซ้อน กวนใจผู้ใช้งาน และเตือนให้เช็คชื่อแม้เป็นวันหยุด:
- **ปัญหา 1: แจ้งเตือนพร่ำเพรื่อ**
  - **การแก้ไข:** แก้ไขไฟล์ `public/js/main.js` เพื่อดักจับ Event เมื่อระบบแสดง Popup สั่งงานด่วน (เช่น คำสั่งให้ครูเช็คชื่อจากแอดมิน) ให้เรียกคำสั่ง `markAsRead()` ทันที เพื่อไม่ให้ Popup เดิมกลับมาแสดงอีกในหน้าอื่นๆ นอกจากนี้ ได้ปรับปรุงเงื่อนไขจุดสีแดง (Red Dot) ที่แถบเมนูด้านซ้าย ให้ตรวจสอบสถานะการอ่านก่อน (`is_read == 0`) หากเคยอ่าน/เห็นแล้ว จุดแดงจะหายไป ไม่กระพริบค้างไว้
- **ปัญหา 2: แจ้งเตือนเช็คชื่อในวันหยุด**
  - **การแก้ไข:** แก้ไขไฟล์ `api/notifications.php` โดยเพิ่มตรรกะการตรวจสอบวันหยุด (ดึงข้อมูลจากตาราง `academic_days` และตรวจสอบวันเสาร์-อาทิตย์) หากพบว่าเป็นวันหยุด ระบบจะข้ามการสร้างคำเตือน "ยังไม่ได้เช็คชื่อโฮมรูมวันนี้" ให้อัตโนมัติ

# Session Log: 8 มิถุนายน 2026

เอกสารฉบับนี้บันทึกการเปลี่ยนแปลงระบบ CNP APP ประจำวันที่ 8 มิถุนายน 2026

---

## 1. ปรับปรุงหน้าเช็คชื่อรายวิชา (`attendance_subject.html`)

### 1.1 ดึงข้อมูลโฮมรูมมาแสดงทุกสถานะโดยอัตโนมัติ

- **ปัญหาเดิม:** เมื่อครูเปิดหน้าเช็คชื่อรายวิชาแต่ยังไม่เคยบันทึก ระบบจะตั้งค่าเริ่มต้นของทุกคนเป็น **"มา"** อัตโนมัติ ซึ่งไม่ถูกต้องถ้าครูที่ปรึกษาบันทึกสถานะอื่นไว้แล้ว (เช่น ขาด ป่วย ลา)
- **การแก้ไข:**
  - แก้ logic ใน `loadStudents()` ให้ดึงข้อมูลการเช็คชื่อโฮมรูม (`type=daily`) ของวันนั้นมาแสดง **ทุกสถานะ** (ม / ข / ป / ก / ส / ด) โดยไม่ default เป็น "มา" อีกต่อไป
  - นำ `remark` จากโฮมรูมมาแสดงในช่องหมายเหตุด้วย
  - เพิ่ม flag `fromHomeroom` เพื่อระบุว่าแถวนั้นดึงมาจากโฮมรูม
  - แถวที่ข้อมูลมาจากโฮมรูม (ยังไม่ได้บันทึกโดยครูรายวิชา) จะมี:
    - พื้นหลังสีฟ้าอ่อน (`#f0f9ff`)
    - Badge "ที่ปรึกษา" ติดข้างชื่อนักเรียน
  - แสดง Toast แจ้งเตือน "ดึงข้อมูลจากโฮมรูมวันนี้มาแสดงแล้ว" เมื่อระบบดึงข้อมูลโฮมรูมสำเร็จ
- **ลำดับความสำคัญ (Priority):**
  1. มีข้อมูลวิชาบันทึกไว้แล้ว → ใช้ข้อมูลวิชา
  2. ไม่มีข้อมูลวิชา + ครูที่ปรึกษาบันทึกโฮมรูมไว้ → ดึงสถานะจากโฮมรูมมาแสดงทั้งหมด
  3. ไม่มีทั้งคู่ → ไม่มีสถานะ (ให้ครูรายวิชาเลือกเอง)

### 1.2 ลบปุ่ม "เพิ่มวิชาใหม่" ออกจากหัวหน้า

- **การเปลี่ยนแปลง:** ลบปุ่ม **"เพิ่มวิชาใหม่"** (สีเขียว) ออกจาก Header ของหน้า เนื่องจากไม่จำเป็น
- เหลือเพียงปุ่ม **"ดูสถิติรายวิชา"** (สีน้ำเงิน) ที่ตำแหน่งเดียวกัน
