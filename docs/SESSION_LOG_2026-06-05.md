## 1. การปรับปรุงระบบจัดเส้นทางเยี่ยมบ้านนักเรียน (Nearest Neighbor Routing based on Road Distance)

ปรับปรุงขั้นตอนการจัดเส้นทางการเดินทางไปเยี่ยมบ้านนักเรียนในระบบผู้ดูแล/ครูผู้รับผิดชอบ (`admin_students.html` และ `views/admin_students.html`) เพื่อให้การเดินทางมีประสิทธิภาพสูงสุดและตรงตามความต้องการของครู:

- **การคำนวณจากระยะทางจริงบนถนน:** เดิมระบบใช้ OSRM Trip API สำหรับแก้โจทย์ Traveling Salesman Problem (TSP) ซึ่งเน้นระยะทางรวมทั้งลูปเป็นวงปิด ส่งผลให้บางครั้งระบบไม่ได้เรียงจุดที่อยู่ใกล้ที่สุดเป็นลำดับถัดไป เราทำการแก้ไขโดย:
  - ใช้ **OSRM Table API** (`https://router.project-osrm.org/table/v1/driving/...`) เพื่อดึงตารางระยะทาง (Distance Matrix) ระหว่างตำแหน่งทั้งหมดแบบจุดต่อจุดตามสภาพถนนจริง
  - นำมาประมวลผลต่อด้วยอัลกอริทึม **Greedy Nearest Neighbor** ในฝั่ง Client: เริ่มต้นที่จุดตั้งต้น จากนั้นหาจุดถัดไปที่ใกล้ที่สุดตามเส้นทางถนนจริงที่ยังไม่ได้ไป แล้วเดินทางต่อไปเรื่อย ๆ จนครบทุกจุด
  - หาก API ขัดข้องหรือถนนเชื่อมไม่ถึง (ไม่มีข้อมูลระยะทาง) ระบบจะใช้ระยะห่างแบบ Haversine (ระยะขจัดทางภูมิศาสตร์) เป็นแผนสำรอง
  - หลังจากได้ลำดับเส้นทางแล้ว จะเรียก **OSRM Route API** เพื่อนำพิกัดที่เรียงลำดับใหม่มาวาดเส้นทาง (Polyline) แสดงบนแผนที่ Leaflet และแสดงผลขั้นตอนเดินทางทีละสเต็ปอย่างชัดเจน

- **จุดเริ่มต้นเดินทางที่ยืดหยุ่น (Custom Route Starting Point):** 
  - เพิ่ม Dropdown เมนูตัวเลือกจุดเริ่มต้นเดินทาง เพื่อให้ครูเลือกตำแหน่งเริ่มต้นได้อย่างยืดหยุ่น:
    1. **โรงเรียนชัยนาทพิทยาคม (เริ่มต้น)**
    2. **พิกัดปัจจุบันของคุณ (GPS)**: ดึงค่าจากเบราว์เซอร์ผ่าน HTML5 Geolocation API
    3. **กำหนดจุดเริ่มต้นบนแผนที่เอง...**: เปิดโหมดให้ครูสามารถคลิกบนตำแหน่งใดก็ได้ในแผนที่เพื่อตั้งเป็นจุดเริ่มเดินทาง โดยจะมีหมุดสีแดง (ลากย้ายตำแหน่งได้) คอยนำทาง
  - ระบบจะคำนวณระยะทางจากจุดเริ่มต้นใหม่ที่ครูเลือกไปยังบ้านนักเรียนคนแรกที่ใกล้ที่สุดทันที

- **ระบบบันทึกสถานะการเช็คอินเยี่ยมบ้าน (Visited Check-in & Exclude Tracking):**
  - ครูสามารถกดบันทึกหรือยกเลิกการเช็คอินว่า "เยี่ยมบ้านนักเรียนเรียบร้อยแล้ว" ได้โดยตรงผ่าน:
    1. ปุ่มเช็คอินใน Popup ของหมุดแผนที่ Leaflet
    2. ปุ่มเช็คอินบนการ์ดแสดงลำดับการเดินทางในแถบด้านซ้าย
  - **การจัดการสถานะ:**
    - นักเรียนที่ยังไม่ได้ไปเยี่ยม: แสดงด้วยหมุดสีน้ำเงินรูปบ้าน
    - นักเรียนที่เยี่ยมแล้ว: เปลี่ยนเป็นหมุดสีเขียวพร้อมเครื่องหมายถูก
    - ข้อมูลการเช็คอินจะถูกบันทึกไว้ใน `localStorage` ของบราวเซอร์ผู้ใช้งานผ่านคีย์ `cnp_visited_students` เพื่อให้ค่าคงอยู่แม้จะรีเฟรชหน้าเว็บหรือกลับมาใช้ใหม่
    - ระบบจะกรองและข้ามการนำพิกัดนักเรียนที่ "เยี่ยมแล้ว" ไปคำนวณในการจัดเส้นทางทริปถัดไปโดยอัตโนมัติ

- **การดึงเบอร์โทรศัพท์ที่ติดต่อได้จริง (Click-to-Call Prioritization):**
  - ปรับปรุงตรรกะการดึงข้อมูลเบอร์โทรศัพท์ของนักเรียนให้ฉลาดขึ้น โดยคัดเลือกตามลำดับความสำคัญ (เบอร์นักเรียน -> เบอร์ผู้ปกครอง -> เบอร์บิดา -> เบอร์มารดา)
  - แสดงผลในรูปแบบลิงก์โทรศัพท์ `<a href="tel:...">` บน Popup แผนที่ และแถบรายการเส้นทางเดินทาง ช่วยให้ครูสามารถกดโทรออกได้ทันทีเมื่อใช้งานผ่านสมาร์ตโฟนระหว่างเดินทางจริง

- **ปุ่มสลับแผนที่ถนน / ดาวเทียม (Street/Satellite Map Switcher):**
  - เพิ่มแถบควบคุม Layer (L.control.layers) บนแผนที่ Leaflet ในหน้าการจัดการเส้นทางแผนที่ภาพรวม (`overviewMap`), แผนที่ปักหมุดแก้ไขตำแหน่งบ้านนักเรียนของครู (`adminMap`), และแผนที่แสดงตำแหน่งในหน้าข้อมูลส่วนตัวนักเรียน (`studentMap`)
  - ครูและผู้ปกครองสามารถสลับดูแผนที่ปกติ (OpenStreetMap) หรือแผนที่ภาพถ่ายดาวเทียม (Esri World Imagery) ได้ผ่านแถบสลับมุมขวาบน เพื่อช่วยในการตรวจสอบที่ตั้งบ้านพักอาศัยของนักเรียนได้อย่างแม่นยำยิ่งขึ้น

## 2. การปรับปรุงระบบแจ้งเตือนของระบบนิเทศ (Supervision Notifications)

แก้ไขข้อผิดพลาดระบบนิเทศ (ระบบประเมินและนิเทศการสอนของครู) ที่ก่อนหน้านี้ไม่สามารถสร้างการแจ้งเตือนส่งไปยังกล่องข้อความของผู้ใช้ได้เนื่องจากข้อผิดพลาดในคำสั่ง SQL (พยายามเขียนลงคอลัมน์ `status` ที่ไม่มีอยู่จริง และไม่ได้ระบุหัวข้อ `title`):

- **การแก้ไขและบูรณาการ `cnp_notify()`:**
  - นำฟังก์ชันสำเร็จรูปของระบบ `cnp_notify($user_id, $title, $message, $type, $link)` มาแทนที่การเขียน SQL INSERT แบบดิบใน 8 ไฟล์ API เพื่อความปลอดภัยและเป็นระเบียบ
  - กำหนดค่า `title` และ `message` ให้ชัดเจนและอ่านเข้าใจง่าย สำหรับส่งแจ้งเตือนไปยังฝ่ายที่เกี่ยวข้อง (คุณครูผู้รับการนิเทศ, ครูคู่หู (Peer), หัวหน้ากลุ่มสาระฯ (Head), และกรรมการวิชาการ (Academic)):
  
- **รายละเอียดการส่งแจ้งเตือนใน 8 ไฟล์ API ที่ปรับปรุง:**
  1. `api/teacher/supervision_book.php` (การจองเวลา): ส่งแจ้งเตือนหาครูผู้จองเมื่อจองสำเร็จ หรืออัปเดตวันเวลา และแจ้งครูคู่หู/หัวหน้ากลุ่มสาระเมื่อถูกเลือกเป็นคณะกรรมการ
  2. `api/admin/supervision_admin.php` (การอนุมัติ/ระบุกรรมการวิชาการ): แจ้งเตือนฝ่ายวิชาการที่ได้รับมอบหมาย และแจ้งครูผู้รับการนิเทศเมื่อแอดมินอนุมัติผลเรียบร้อย
  3. `api/teacher/supervision_upload_docs.php` (การอัปโหลดเอกสารแผนการสอน): แจ้งครูผู้ส่งเมื่ออัปโหลดไฟล์สำเร็จ และหากอัปโหลดเอกสารครบทั้ง 4 ไฟล์แล้ว จะแจ้งเตือนคณะกรรมการนิเทศทั้ง 3 ท่าน (Peer, Head, Academic) ให้เข้ามาประเมินแผนการสอน
  4. `api/teacher/supervision_evaluate.php` (การทำประเมินเอกสาร/ชั้นเรียน): ส่งผลแจ้งเตือนให้ครูผู้รับการนิเทศทราบเมื่อผลการประเมินแผนการสอน หรือผลการประเมินการสอนในชั้นเรียนได้รับการบันทึกเรียบร้อย
  5. `api/teacher/supervision_post_teach.php` (การบันทึกหลังสอน): แจ้งเตือนครูผู้สอนเมื่อบันทึกแบบสะท้อนคิดหลังสอนเสร็จสมบูรณ์ และส่งแจ้งเตือนหาคณะกรรมการเพื่อปิดงาน
  6. `api/teacher/supervision_cancel.php` (การยกเลิกการจอง): แจ้งเตือนฝ่ายครูผู้จอง และส่งคำยกเลิกแจ้งไปยังกรรมการที่เกี่ยวข้องทั้งหมดเพื่อรับทราบ
  7. `api/public_relations.php` (การอนุมัติข่าวประชาสัมพันธ์): ปรับปรุงการส่งข้อความแจ้งเตือนหาผู้เขียนข่าวเมื่อข่าวได้รับการอนุมัติ/เผยแพร่
  8. `api/notifications.php` (ระบบแจ้งเตือนส่วนกลาง): เพิ่มระบบแจ้งอัปเดตระบบประจำวัน (System Update alert) ของวันที่ 5 มิ.ย. 2569 เพื่อให้แสดงรายละเอียดฟีเจอร์ใหม่แก่ผู้ใช้ทุกคน และเมื่อคลิกปิดแล้วจะถูกซ่อนทันทีไม่ปรากฏซ้ำอีก
