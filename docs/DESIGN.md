# UI/UX & Design Language (CNP APP)

เอกสารฉบับนี้รวบรวมมาตรฐานการออกแบบ (Design System), โทนสี (Color Palette), ฟอนต์ (Typography), และแนวทางการสร้าง User Interface ของระบบ CNP APP เพื่อให้การพัฒนาหน้าเว็บใหม่ ๆ เป็นไปในทิศทางเดียวกันและคงความสวยงามระดับพรีเมียม

---

## 🎨 1. สีและธีมหลัก (Color Palette & Theme)

ระบบใช้แนวคิด **Dynamic Theming** โดยอิงจาก "คณะสี" (House) ของนักเรียนเป็นหลัก ควบคู่ไปกับสีพื้นฐานที่ให้ความรู้สึกเป็นทางการ ทันสมัย และอ่านง่าย

### ฐานสีหลัก (Base Colors)
- **Primary Navy:** `#0f172a` (ใช้เป็นสีปุ่มหลัก, Header พื้นหลัง, หัวข้อสำคัญ)
- **Background:** `#f1f5f9` (สีพื้นหลังหลักของหน้าเว็บ ลดความสว่างจากสีขาวปกติเพื่อให้สบายตา)
- **Text Dark:** `#1e293b` (สีข้อความทั่วไป)
- **Text Muted:** `#64748b` หรือ `#475569` (สีข้อความรอง, Label ฟอร์ม)

### สีประจำคณะ (House Colors)
สไตล์ของระบบจะถูกจัดการผ่าน CSS Variables (เช่น `--house-color`) ซึ่งจะเปลี่ยนตามข้อมูลคณะของนักเรียน:
1. 💗 **ขุนสรรค์ (Pink):** `#db2777`
2. ❤️ **ขุนศรี (Red):** `#dc2626`
3. 💚 **เจ้ายี่ (Green):** `#16a34a`
4. 💛 **ธรรมจักร (Yellow):** `#ca8a04`

## 🔤 2. ตัวอักษร (Typography)

เพื่อความทันสมัยและอ่านง่าย ระบบใช้ Web Fonts จาก Google Fonts:

- **Google Sans** / **Inter**: เป็นฟอนต์หลักสำหรับข้อความทั้งหมด (น้ำหนัก 400 สำหรับเนื้อหาทั่วไป, 500 สำหรับ Label, 700/800/900 สำหรับหัวข้อที่ต้องการเน้น)
- **ขนาดตัวอักษร:** เน้นความชัดเจนและกะทัดรัด
  - หัวข้อใหญ่ (H4/H5): `1.05rem` ถึง `1.3rem` แฝงความหนาแบบ `fw-black`
  - เนื้อหาทั่วไป: `0.92rem` ถึง `1rem`
  - ข้อความรอง (Small/Badge): `0.75rem` ถึง `0.85rem`

## 📐 3. โครงสร้าง Layout และ Components (UI Architecture)

### 📱 3.1 Mobile-First & Grid
- ออกแบบบนรากฐานของ **Bootstrap 5.3**
- ใช้ระบบ **Grid (`row g-3`, `g-4`)** เพื่อจัดระเบียบช่องว่าง (Spacing) ให้เท่ากันและตอบสนองได้ดีบนทุกหน้าจอ
- แบ่งเลย์เอาต์หน้าจอใหญ่เป็น 2 คอลัมน์หลัก เช่น 3:9 (Badge Profile ด้านซ้าย, ข้อมูล Detail ด้านขวา)

### 💳 3.2 Cards & Containers
- **`.card-modern`**: ขอบมน (Border Radius) 20px, ไม่มีเส้นขอบ (border: none), มีเงาอ่อน ๆ แบบ Soft Shadow (`box-shadow: 0 8px 24px rgba(0,0,0,0.05)`)
- **`.section-title`**: มีการจัดกลุ่มหัวข้อโดยใช้ไอคอน (Icon Box) สีพื้นหลังจาง ๆ ตัดกับสีของไอคอน เพิ่มมิติทางสายตา

### 🗂️ 3.3 การจัดกลุ่มเนื้อหา (Group Dividers)
- แทนที่จะใช้ระบบแท็บ (Tabs) ซึ่งอาจทำให้ดูข้อมูลได้ยากบนมือถือ ระบบใช้การแสดงผลแบบยาว (Scrollable) 
- ใช้ **Group Divider** (ไอคอน + ชื่อกลุ่ม + เส้นแบ่งแบบ Gradient) เป็นตัวแบ่งหมวดหมู่ข้อมูล เพื่อให้ผู้ใช้รับรู้ได้ง่าย

### 🔘 3.4 Form Controls & Buttons
- **Inputs & Selects**: ขอบมน 12px, มี padding ที่พอดี, และมี Transition ตอน Focus (ขอบสี Navy พร้อม shadow)
- **Radio/Toggle (Button Group)**: แทนที่ Dropdown ที่มีตัวเลือกน้อยด้วย ปุ่มสลับ (Toggle Buttons `.btn-group-modern`) เพื่อลดจำนวนคลิกและทำให้ UI ดูเป็น Interactive App มากขึ้น
- **View/Edit Mode**: ฟอร์มเริ่มต้นในโหมดดูข้อมูล (View Mode) จะทำการซ่อนขอบ Input เพื่อให้ดูเหมือนเป็นข้อความธรรมดา (Text) หากกดแก้ไข (Edit Mode) อินพุตจะกลับมาพร้อมให้พิมพ์

## ✨ 4. สุนทรียภาพและการตอบสนอง (Aesthetics & Micro-animations)

- **Hover Effects:** ปุ่มหลักและการเลือกคณะสี (`.house-option`) มีการตอบสนองเมื่อเอาเมาส์ชี้ เช่น ขยายตัว (Scale 1.2) หรือเปลี่ยนสีพื้นหลัง เพื่อเพิ่มความมีชีวิตชีวา (Dynamic Design)
- **ID Badge Header:** หน้าโปรไฟล์มีการประยุกต์ใช้พื้นหลังลายตาราง (Background Image) ผสมผสานกับการใช้ `mix-blend-mode: multiply` ซ้อนทับด้วยสีประจำคณะ เกิดเป็นดีไซน์การ์ดประจำตัวที่โดดเด่นและพรีเมียม
- **Icons:** ใช้ **Bootstrap Icons** เสมอ เพื่อความสม่ำเสมอในสไตล์ของกราฟิก
- **SweetAlert2:** แทนที่ Alert ธรรมดาด้วย Popup ที่มีดีไซน์สอดคล้องกับหน้าเว็บ (ใช้ customClass `rounded-4`)

## 🚀 5. สรุปความประทับใจ (The WOW Factor)

เป้าหมายของ CNP APP คือการ **WOW** ผู้ใช้งานตั้งแต่แรกเห็น ด้วยการปฏิเสธดีไซน์แบบระบบราชการแบบเก่า:
1. สีไม่จืดชืด และมีการใช้เงาเพื่อให้เกิดมิติ
2. อินเทอร์เฟซตอบสนอง (Responsive) และมี Feedback ทันทีเมื่อโต้ตอบ
3. พื้นที่ว่าง (Whitespace) ถูกนำมาใช้ประโยชน์ในการแยกหมวดหมู่ข้อมูล ไม่ให้ดูรกหรือแน่นจนเกินไป
