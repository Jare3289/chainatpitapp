- **ลำดับการลบ** (FK constraint ต้องลบ child tables ก่อน):
  1. `DELETE FROM attendance WHERE student_id IN (SELECT id FROM students WHERE student_id IN (...))`
  2. `DELETE FROM point_transactions WHERE student_id IN (...)`
  3. `DELETE FROM student_evaluations WHERE student_id IN (...)`
  4. `DELETE FROM attendance_subjects WHERE student_id IN (...)` ← พบ FK ใหม่ระหว่างทำ

### 21. Teacher Dropdown Mismatch and Button Style Fix
- **ปัญหา:** 
  1. ปุ่มสลับโหมดในตัวเลือกตารางสอนแสดงผลเพี้ยน (มีขอบหรือสีพื้นหลังสีเทาโผล่ขึ้นมาตามการเรนเดอร์ของ Bootstrap) บนบางเบราว์เซอร์
  2. เมื่อแอดมินหรือครูเลือกครูผู้สอนในตัวกรองแล้ว ตารางสอนไม่ดึงข้อมูลมาแสดงผล (ไม่มา)

  1. ปุ่ม Inactive ไม่ได้ใส่คลาสล้างขอบและล้างพื้นหลังให้โปร่งใส (`bg-transparent border-0`) อย่างชัดเจน ทำให้ตกค้างขอบและพื้นหลังของ Bootstrap ดั้งเดิม
  2. ระบบฐานข้อมูลตารางเรียน/สอนใช้รหัสอ้างอิง `t.teacher_id` เป็น Primary Key ของครูในตาราง `teachers` (`teachers.id`) แต่ API ตัวเลือกรายชื่อครู (`api/get_teachers_list.php`) ดึงเฉพาะฟิลด์ `user_id` มาและนำมาส่งเป็นฟิลด์ `id` ส่งผลให้เมื่อหน้าบ้านนำ `user_id` ไปค้นหาตารางสอนในหลังบ้าน ข้อมูลจึงไม่ถูกจับคู่และไม่มีรายวิชาใดส่งกลับมา (ข้อมูลไม่มา)
  3. ในหน้า `views/admin_timetable.html` ชื่อคุณครูในตัวเลือกแสดงผลเป็น `undefinedundefined undefined` เนื่องจากไปแมปข้อมูลจากฟิลด์เดี่ยวไม่มีอยู่ใน API โครงสร้างใหม่

  - **API Expansion:** อัปเดต `api/get_teachers_list.php` ให้คืนค่าทั้ง `teachers_id` (Primary Key จากตาราง `teachers` สำหรับงานตารางสอน) และ `id`/`user_id` (สำหรับงานบันทึกความดี/สาธารณประโยชน์) เพื่อป้องกันผลกระทบย้อนหลังต่อส่วนอื่นของระบบ (Backward Compatibility)
  - **ID Value Mapping:** แก้ไขการแมปค่าในไฟล์ `views/timetable.html` และ `views/admin_timetable.html` ให้ส่งตัวเลือกค่าคุณครูเป็น `teachers_id` ที่ถูกต้อง พร้อมกับใช้คุณสมบัติ `.name` ที่ส่งมาจาก API ในการเรนเดอร์ชื่อเต็ม เพื่อแก้ปัญหาค่า `undefined` หายไปและแสดงชื่อจริงได้อย่างสวยงาม 100%

### 22. Class Import Parameter Alignment and Dry Run Duplicates Overwrite Support
- **ปัญหา:**
  1. เมื่อแอดมินพยายามอัปโหลดไฟล์ห้องเรียนผ่าน `admin_classes.html` ระบบแจ้งข้อผิดพลาดว่า **"ไม่พบไฟล์ที่อัปโหลด"**
  2. ระบบห้องเรียนในอดีตไม่มีกระบวนการตรวจสอบค่าซ้ำ (Dry Run Preview) และไม่มีหน้าต่างแสดงความแตกต่างของคอลัมน์ข้อมูลชุดใหญ่ก่อนทำการยืนยันอัปเดตทับข้อมูลเก่า
- **สาเหตุ:**
  1. ปลั๊กอินแชร์ `importModal.js` ส่งไฟล์อัปโหลดผ่านตัวแปรฟอร์มชื่อ `excelFile` แต่ไฟล์ `api/admin/upload-classes.php` ดั้งเดิมตั้งรับผ่านตัวแปรชื่อ `file` จึงเกิดการมองไม่เห็นไฟล์กัน
  2. ตัว API นำเข้าห้องเรียนไม่ได้ประมวลผลพารามิเตอร์ `dry_run` และไม่มีการเขียนอาเรย์ `duplicates` สำหรับเปรียบเทียบข้อมูลเดิมในระบบกับข้อมูลใหม่จากไฟล์อัปโหลด
- **แก้ไข:**
  - **Fallback Input Matching:** แก้ไข `api/admin/upload-classes.php` ให้รองรับทั้งตัวแปร `excelFile` และ `file` (`$_FILES['excelFile'] ?? $_FILES['file']`) ทำให้สามารถสื่อสารและอัปโหลดข้อมูลร่วมกับระบบกลางได้อย่างสมบูรณ์แบบ
  - **Premium Dry Run & Duplicate Checker:** พัฒนาระบบเปรียบเทียบข้อมูลห้องเรียนขั้นสูงขึ้นมา โดยดึงข้อมูลเดิมจากฐานข้อมูล (`grade_level`, `building`, `floor`, `room_no`, `program`) มาเปรียบเทียบค่าความแตกต่างกับไฟล์ที่อัปโหลดทีละคอลัมน์
  - **Dynamic Step-by-Step Update:** หากเลือกตรวจสอบแบบ Dry Run ระบบจะส่งข้อมูลกลับไปแสดงเป็นตารางสถิติสรุป (จำนวนข้อมูลเพิ่มใหม่ / รายการที่จะได้รับอัปเดตข้อมูลชุดใหญ่ทับลงไป) และเมื่อกด "ยืนยันนำเข้าข้อมูล" ระบบจะทำธุรกรรม (Transaction) เขียนทับและอัปเดตข้อมูลห้องเรียนทุกคอลัมน์ทันทีตามที่แอดมินต้องการอย่างแม่นยำและเสถียรสูงสุด!

### 23. Timetable Dynamic Department-Teacher Filtering and Spacing Alignment
- **ปัญหา:**
  1. ปุ่มและกลุ่มตัวเลือกวิชาในระบบตารางเรียน (`views/timetable.html`) บางกลุ่มสาระการเรียนรู้กดแล้วไม่มีข้อมูลตารางเรียนแสดงผล (เช่น สังคมศึกษาฯ และ กิจกรรมฯ)
  2. ตัวเลือกกล่องครูผู้สอน (`select#filterTeacher`) แสดงคุณครูทั้งหมดของโรงเรียนรวมกัน ทำให้แอดมินหรือผู้ใช้ใช้งานยากเมื่อเลือกกลุ่มสาระย่อยไปแล้ว แต่รายชื่อครูไม่ได้รับการคัดกรองตามกลุ่มสาระนั้นๆ
- **สาเหตุ:**
  1. ค่าแผนก/กลุ่มสาระในปุ่มหน้าบ้านมีช่องว่าง เช่น `'สังคมศึกษา ศาสนา และวัฒนธรรม'` และ `'กิจกรรมพัฒนาผู้เรียน'` แต่ในฐานข้อมูลคอลัมน์ `department` สะกดเป็น `'สังคมศึกษาศาสนาและวัฒนธรรม'` (ไม่มีช่องว่าง) และใช้คำว่า `'แนะแนว'` เป็นต้น ทำให้การสอบถาม SQL คืนค่าว่างเปล่า
  2. ข้อมูลคุณครูที่ดึงผ่าน `api/get_teachers_list.php` ในอดีตไม่มีคอลัมน์ `department` ถูกดึงมาด้วย และหน้าบ้านยังไม่มีฟังก์ชันจำแนกครูตามกลุ่มสาระการเรียนรู้แบบ Real-time
- **แก้ไข:**
  - **Database Spacing Harmonization:**
    - ปรับปรุง `api/timetable.php` ฝั่งหลังบ้านให้ฉลาดขึ้น โดยทำการแทนที่และตัดช่องว่าง (`str_replace(' ', '', ...)`) เพื่อค้นหาข้อมูลกลุ่มสาระ `'สังคมศึกษาศาสนาและวัฒนธรรม'` ได้อย่างยืดหยุ่น 100%
    - เพิ่มการแมปค่าสาระกลุ่ม `'บริหาร'` ให้ดึงตำแหน่ง `'ผู้อำนวยการ'` และ `'รองผู้อำนวยการ'` ออกมารวมกันได้อย่างชาญฉลาด
  - **Dropdown and Quick Filters Sync:**
    - อัปเดตไฟล์ `views/timetable.html` ให้ส่งค่า `value="สังคมศึกษาศาสนาและวัฒนธรรม"` แต่แสดงข้อความที่มีช่องว่างอย่างสวยงาม
    - เปลี่ยนปุ่มกลุ่มสาระกิจกรรมพัฒนาผู้เรียนเป็นกลุ่มสาระ `'แนะแนว'` เพื่อให้ตรงกับมาตรฐานข้อมูลจริงในฐานข้อมูลอย่างสมบูรณ์แบบ
  - **Dynamic Teacher Cascade Filter:**
    - ดึงคอลัมน์ `department` จากตาราง `teachers` เข้ามาใน `api/get_teachers_list.php`
    - พัฒนาฟังก์ชันหน้าบ้าน `filterTeachersByDept()` ให้คัดกรองรายชื่อคุณครูใน `select#filterTeacher` ทันทีเมื่อผู้ใช้งานคลิกปุ่ม Quick Filter หรือเลือกกล่องเปลี่ยนกลุ่มสาระ เพื่อการใช้งานที่ลื่นไหลระดับไฮเอนด์!

- **ปัญหา:** ตารางสอนของคุณครูบางท่านปรากฏวิชาสอนทับซ้อนกันในคาบและเวลาเดียวกัน (เช่น สอนวิชาคณิตศาสตร์ในห้อง 246 และสอนวิชาคอมพิวเตอร์ในห้อง 922 ในคาบที่ 1 ของวันจันทร์พร้อมกัน) ซึ่งเป็นไปไม่ได้ในทางปฏิบัติ
- **สาเหตุการซ้ำซ้อน:**

  - นอกจากนี้ ตารางวิชาการงานอาชีพ/คอมพิวเตอร์ (Rows 671-700) ถูกนำเข้าผิดพลาดและผูกไว้กับ `teacher_id = 65` แทนที่จะเป็นครูคอมพิวเตอร์ที่แท้จริง
- **การแก้ไขและล้างข้อมูลเชิงโครงสร้าง:**
  - พัฒนาและรันสคริปต์ `scratch/fix_timetable_duplicates.php` เพื่อล้างข้อมูลระดับ Database:
    1. ลบรายการตารางเรียนวิชาคณิตศาสตร์ที่ซ้ำซ้อนทับกันทั้งหมดของ `teacher_id = 65` (Rows 3737-3756) ออกจากระบบ
    2. ย้ายและโอนข้อมูลตารางสอนวิชาการงาน/คอมพิวเตอร์ (Rows 671-700) จาก `teacher_id = 65` ไปยัง `teacher_id = 181` (**นางสาวเพ็ญนิการ์ ยอดเกิด** คุณครูกลุ่มสาระการงานอาชีพที่เป็นเจ้าของห้องและตารางจริงที่เดิมไม่มีข้อมูลคาบสอนในระบบ)
    3. ลบโปรไฟล์ครูที่ซ้ำซ้อน `id = 65` ออกจากระบบอย่างสมบูรณ์
    4. อัปเดตรหัสครูหลัก `id = 60` (นางพรรณิภา อนันทสุข) ให้มีรหัสครูอ้างอิงอย่างถูกต้องเป็น `122` เพื่อความถูกต้องในโครงสร้างข้อมูล
  - **ผลลัพธ์:** การตรวจสอบด้วยสคริปต์ตรวจความซ้ำซ้อนระบุชัดเจนว่า **ไม่พบข้อมูลตารางเรียนซ้ำซ้อนหรือทับคาบเวลา (Overlap / Double-Booking) ในฐานข้อมูล CNP APP อีกเลย (Found 0 duplicate allocations)** โครงสร้างข้อมูลตารางเรียนเสถียร สมบูรณ์ และทำงานได้ถูกต้อง 100%!

### 31. Teacher/Admin 10-Period View Limit (จำกัดการแสดงตารางสอนที่ 10 คาบ)
- **ปัญหา:** หน้าตารางสอนของครูและแอดมินแสดงจำนวนคาบเรียนมากเกินไป (12 คาบ: คาบ 0 - คาบ 11) ซึ่งคาบ 0 (โฮมรูม) และคาบ 11 ไม่จำเป็นต้องแสดงในตารางหลัก ทำให้อ่านยากและไม่กระชับ

### 32. Overlapping Timetable Fix — "All Teachers" Grid Overlap Prevention
- **ปัญหา:** ผู้ใช้รายงานว่าครูบางคนดูเหมือนสอนคาบเดียวกัน 2 วิชา ซึ่งจากการตรวจสอบ Database เชิงลึกยืนยันว่า **ในฐานข้อมูลไม่มีรายการคาบสอนซ้ำซ้อนเลยแม้แต่รายการเดียว (0 overlaps)**
- **สาเหตุของบั๊กแสดงผล:**
  - เกิดจากการที่แผงควบคุมมีปุ่ม **"ครูทุกคน"** ในโหมดตารางสอน เมื่อแอดมินคลิกปุ่มนี้หรือเมื่อระบบเริ่มต้นโดยไม่มีการกรอง ระบบจะดึงตารางสอนของ **"ครูทุกคนในโรงเรียน" (50+ คน)** มาพยายามวาดลงบนตาราง 7x10 ช่องเดียว!
  - ส่งผลให้ในแต่ละช่องแสดงวิชาและคุณครูทั้งหมดของทั้งโรงเรียนปะปนกัน ทำให้อ่านแล้วเหมือนครูคนเดียวกันสอนซ้อนกัน 2 วิชา ซึ่งเป็นข้อผิดพลาดด้าน UX (UX Flaw)
- **แก้ไข `views/timetable.html`:**
  - **ลบปุ่ม "ครูทุกคน" ออกจากระบบตัวกรองการสอนรายครู** เนื่องจากตารางสอนรายครูออกแบบมาให้ดูทีละคนเพื่อความเป็นระเบียบและถูกต้อง
  - **ระบบ Auto-Select คนแรก:** หากผู้ใช้เปลี่ยนกลุ่มสาระหรือล้างการกรอง ระบบจะเลือกคุณครู **"คนแรกในรายการตัวกรอง"** โดยอัตโนมัติ (แทนที่จะปล่อยว่างเป็นครูทุกคน) ทำให้หน้าจอโหลดตารางเรียนของคุณครูคนนั้นขึ้นมาแสดงอย่างคลีน สะอาด และถูกต้อง 100% ไม่มีวิชาทับซ้อนหรือหน้าจอว่างเปล่าอีกต่อไป

### 36. Smart PR Hub Interface Redesign & Dynamic Interaction Suite (ปรับดีไซน์และระบบอินเทอร์แอคทีฟประชาสัมพันธ์อัจฉริยะ)
- **ปัญหา:** หน้าข่าวประชาสัมพันธ์ (`public_relations.html`) ในเวอร์ชันแรกมีโครงสร้างและตัวกรองที่เรียบง่ายเกินไป ไม่สอดคล้องกับหน้าเว็บพรีเมียมอื่นๆ ที่ปรับเปลี่ยนโหมดโครงสร้างเมนูแบบใหม่ และยังขาดคุณสมบัติเชิงโต้ตอบที่เป็นประโยชน์ในชีวิตจริง
- **การดำเนินการปฏิรูปและยกระดับเลย์เอาต์:**
  - **1. โครงสร้างเชลล์เมนูด้านข้างตัวใหม่ (Shell Sidebar Parity):** ปรับโครงสร้าง Wrapper ของ `public_relations.html` จากรูปแบบเก่าของ AdminLTE (ที่ใช้ `#navbar-container` และ `#sidebar-container` พร้อมเรียก Sidebar ผิดประเภท) ให้หันมาใช้งานโครงสร้าง Custom Grid ระดับพรีเมียมตัวใหม่ เช่นเดียวกับหน้าตารางสอนและหน้าสำคัญอื่นๆ (`wrapper` -> `sidebar#mainSidebar` -> `main-panel` -> `#header-container` -> `app-content`) พร้อมแก้บั๊กการส่งอาร์กิวเมนต์เข้า `renderSidebar(role, user)` ส่งผลให้ระบบเมนูแสดงผลสมบูรณ์แบบ ทันสมัย และไม่มีจุดสะดุดอีกต่อไป
  - **2. แถบประกาศด่วนพิเศษแบบวิ่งได้ด้วย CSS3 (High-Performance Hardware-Accelerated CSS Ticker Loop):** ยกเลิกและลบการใช้งานแท็กโบราณ `<marquee>` ที่ไม่มีประสิทธิภาพในการประมวลผลบนบราวเซอร์รุ่นใหม่ แล้วเปลี่ยนมาพัฒนาโครงสร้างแบบพรีเมียมด้วยคอนเทนเนอร์ `.ticker-wrapper` ร่วมกับคลาสอนิเมชัน `.ticker-content` ที่ควบคุมผ่าน CSS3 Keyframes เคลื่อนที่แบบ Hardware-Accelerated ราบรื่นดุจแพรไหม (Silky Smooth Scrolling) พร้อมสนับสนุนการหยุดวิ่งชั่วคราวเมื่อผู้ใช้นำเมาส์ไปชี้ (Pause on Hover) เพื่อให้ผู้ใช้สามารถอ่านข่าวด่วนที่สุดของโรงเรียนได้อย่างสะดวกสบายสูงสุด
  - **3. กล่องค้นหาอัจฉริยะ (Live Fuzzy Search Filter):** เพิ่มกล่องสแกนคำค้นหาแบบเรียลไทม์ ช่วยให้ผู้ใช้สามารถพิมพ์เพื่อสืบค้นหัวข้อประกาศ เนื้อหาภายใน หรือกระทั่งชื่อผู้เขียนได้อย่างลื่นไหลและแสดงผลทันทีโดยไม่ต้องกดโหลดหน้าใหม่
  - **4. ปุ่มไลก์หัวใจและแอนิเมชัน Emoji ลอยตัว (Micro-Interaction Hearts Burst):** เพิ่มระบบปุ่มหัวใจในฟูตเตอร์ของการ์ดข่าวสาร เมื่อคลิกจะเกิดอนิเมชันหัวใจลอยพุ่งขึ้นมาจากขอบปุ่มแบบไดนามิกพร้อมนับจำนวนไลก์สะสม และจัดเก็บข้อมูลอย่างปลอดภัยใน Browser Storage ของผู้ใช้
  - **5. ปุ่มปักหมุดข่าวที่สนใจ (Bookmarked / Pinned Posts Shelf):** ผู้ใช้สามารถกดไอคอนปักหมุดที่มุมการ์ดเพื่อเก็บโพสต์ที่ชอบหรือสำคัญไว้ในรายการโปรดส่วนตัว ซึ่งจะมีตัวกรองพิเศษ "ปักหมุดของฉัน" พร้อมป้ายแสดงจำนวนที่ปักหมุดอยู่ ช่วยให้ตามอ่านประกาศสำคัญย้อนหลังได้อย่างรวดเร็วและเป็นประโยชน์ในชีวิตจริงสูงสุด!
  - **6. หน้าจอรอโหลดแบบ Skeleton Shimmer Card:** เพิ่มโครงสร้าง Skeleton Loader ที่แสดงแสงวิ่งแบบ Gradient เงา เพื่อพรีวิวโครงร่างก่อนที่ข้อมูล AJAX จะเรนเดอร์เสร็จสิ้นอย่างสมบูรณ์ ป้องกันการกระตุกของหน้าจอ
  - **7. การปฏิบัติตามมาตรฐานระบบโทเค็นและส่วนประกอบส่วนกลาง (Design Tokens & Component Alignment):** ปฏิรูปโครงสร้างคลาสสไตลิงและสีสันของ `public_relations.html` ให้รองรับการทำงานกับระบบดีไซน์หลักอย่างครบถ้วน:
    - ปรับสีแบนเนอร์ส่วนหัว (Hero Banner) และธีมสีให้ใช้โทนสีแบรนด์เนมน้ำเงิน-ฟ้าของโรงเรียน `bg-gradient-navy` (ใช้ตัวแปร `--navy-700` และ `--navy-500`) เพื่อความสวยงามกลมกลืนกับหน้าตารางสอน
    - ปรับการ์ดข่าวสารให้เรียกใช้งานคอมโพเนนต์ส่วนกลาง `.cnp-card` และคลาส `.cnp-card--compact` ร่วมกับ CSS variables ช่วยควบคุมระยะห่าง (Spacing) และมุมโค้งมนตามมาตรฐานระบบหลัก
    - แปลงแถบประเภท/หมวดหมู่เป็นระบบ Badge มาตรฐาน `.cnp-badge`, `.cnp-badge--danger` (ประกาศสำคัญ), `.cnp-badge--info` (วิชาการ), และ `.cnp-badge--warning` (กิจกรรม)
    - นำเข้าคอมโพเนนต์แสดงกล่องข้อมูลว่างเปล่า `.cnp-empty` ร่วมกับ `.cnp-empty__icon` และ `.cnp-empty__text`
    - ล้างคลาส Shimmer ทับซ้อนทิ้งแล้วสลับมาใช้คลาสประมวลผลกระดูกงูแบบพิกัดสีอย่างมีเอกภาพร่วมกับ `.cnp-skeleton`
    - การันตีการรองรับระบบจัดหน้าของ Bootstrap 5 อย่างสมบูรณ์แบบโดยไม่มีการใช้งานคลาสจัดระยะทางลัดนอกเหนือมาตรฐาน

    - ออกแบบสีเส้นขอบการ์ดของแต่ละหมวดหมู่ (Category Border Top) ให้ใช้โทนสีแบรนด์เนมอย่างสวยงามกลมกลืน
  - **2. เพิ่มระบบกำหนดสิทธิ์การเข้าชมข่าวสาร (Target Audience Visibility Constraints):**
    - อัปเดตฐานข้อมูลตาราง `public_relations` โดยการเขียนและรันสคริปต์ไมเกรชันเพื่อเพิ่มคอลัมน์ `visibility` (มีค่าที่เป็นไปได้คือ `all`, `teacher`, `student`)
    - เพิ่มดรอปดาวน์ **"กลุ่มเป้าหมาย (การมองเห็น)"** ภายในฟอร์มสร้างโพสต์ประชาสัมพันธ์ เพื่อให้ครู นักเรียน หรือแอดมินระบบสามารถระบุกลุ่มเป้าหมายก่อนส่งข่าวได้ทันที
    - ปรับแต่งคำสั่ง SQL ใน `api/public_relations.php` ให้กรองความปลอดภัยอย่างเข้มงวด โดยผู้ใช้ในบทบาทต่างๆ จะมองเห็นเฉพาะข่าวสารที่ได้รับการอนุมัติแล้วและตรงกับสิทธิ์การเข้าชมของตนเอง (หรือเห็นโพสต์ที่ตนเองเป็นคนสร้าง) ในขณะที่ผู้ดูแลระบบ (Admins) สามารถมองเห็นโพสต์ข่าวสารทั้งหมดได้เสมอเพื่อตรวจสอบความเรียบร้อย
    - แทรกการแสดงผลป้ายแสดงกลุ่มเป้าหมาย (`เฉพาะครู`, `เฉพาะนักเรียน`, `ทุกคน`) บนการ์ด Feed ข่าวสารประชาสัมพันธ์ และภายในหน้าต่างแสดงรายละเอียดข่าวสารแบบป๊อปอัป (Post Details Preview Dialog) เพื่อความชัดเจนสูงสุด
